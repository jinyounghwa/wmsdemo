# SKILL: 재고 현황 `/inventory`

## 목적
품목별·위치별 재고를 조회하고 재고 상태를 시각적으로 파악하는 데모.

## 레이아웃 구성

### 상단 필터/검색
- 검색: 품목코드, 품목명
- 필터: 카테고리, 창고구역(Zone), 재고상태(정상/부족/초과/불량)
- 뷰 전환: 테이블 뷰 / 카드 뷰 (토글)

### 재고 요약 카드 (4개)
- 전체 SKU 수
- 정상 재고
- 부족 재고 (안전재고 미달)
- 불량/격리 재고

### 테이블 뷰
| 컬럼 | 설명 |
|------|------|
| 품목코드 | SKU-0001 형식 |
| 품목명 | 상품명 |
| 카테고리 | 전자부품 등 |
| 위치 | Zone-Rack-Bin (예: A-03-05) |
| 현재고 | EA |
| 안전재고 | EA |
| 상태 | 뱃지 |
| 마지막 이동 | 날짜 |

### 카드 뷰
품목 카드 그리드 (3~4열)
- 카드: 품목명 + 코드 + 현재고 크게 + 안전재고 대비 프로그레스 바 + 상태 뱃지

### 재고 상세 사이드패널 (행 클릭 시)
- 품목 기본정보
- 위치별 재고 분포 (동일 SKU가 여러 위치에 있을 경우)
- 최근 입출고 이력 (5건)
- 입고/출고 버튼 (→ 각 관리 페이지 이동)

### 창고 적재율 시각화 (선택)
- Zone별 적재율 바 차트
- 전체 창고 용량 대비 사용률

## Mock 데이터 예시
```ts
interface InventoryItem {
  sku: string           // 'SKU-0001'
  name: string
  category: string
  zone: string          // 'A', 'B', 'C'
  rack: string
  bin: string
  currentQty: number
  safetyQty: number     // 안전재고
  status: 'normal' | 'low' | 'excess' | 'defect'
  lastMovedAt: string
}
// 50~80건 생성 (faker 또는 하드코딩)
```

## 디자인 포인트
- 재고 상태별 색상: 정상 green, 부족 red, 초과 blue, 불량 gray
- 프로그레스 바: 안전재고 대비 현재고 비율 시각화
- 사이드패널: 오른쪽에서 슬라이드인 애니메이션
- 테이블/카드 뷰 전환 시 부드러운 transition
