# SKILL: 입고 관리 `/inbound`

## 목적
입고 예정 목록 조회 및 입고 처리 흐름 데모. 상태 변경과 검수 입력을 시뮬레이션.

## 레이아웃 구성

### 상단 필터/액션 바
- 검색: 발주번호, 거래처명
- 필터: 상태(전체/입고예정/검수중/완료/불량)
- 날짜 범위 선택
- 버튼: "입고 등록" (모달 오픈)

### 입고 목록 테이블
| 컬럼 | 설명 |
|------|------|
| 발주번호 | PO-20240001 형식 |
| 거래처 | 공급업체명 |
| 품목 | 대표 품목명 + 외 N건 |
| 예정수량 | EA |
| 실입고수량 | 검수 후 입력값 |
| 입고예정일 | 날짜 |
| 상태 | 뱃지 (색상 구분) |
| 액션 | 검수 버튼 / 상세 버튼 |

### 상태 뱃지 색상
- 입고예정: blue
- 검수중: yellow
- 완료: green
- 불량/불일치: red

### 입고 상세 모달
- 발주 정보 (번호, 거래처, 날짜)
- 품목 리스트: 품목코드, 품목명, 발주수량, 실입고수량(입력), 불량수량(입력)
- 비고 입력
- 버튼: 검수 완료 → 상태를 "완료"로 변경 (로컬 상태 업데이트)

### 입고 등록 모달
- 거래처 선택 (드롭다운, Mock 데이터)
- 품목 추가 (+ 버튼으로 행 추가)
- 입고 예정일 선택
- 저장 시 목록에 추가 (Zustand)

## Mock 데이터 예시
```ts
interface InboundOrder {
  id: string         // 'PO-20240001'
  vendor: string
  items: { sku: string; name: string; qty: number }[]
  scheduledDate: string
  status: 'scheduled' | 'inspecting' | 'completed' | 'defect'
}
// 20건 정도 생성
```

## 디자인 포인트
- 테이블 행 hover 시 배경 강조
- 모달은 오버레이 + 중앙 카드 형태
- 상태 변경 시 즉각 UI 반영 (낙관적 업데이트)
